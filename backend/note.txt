=================================================================================================
Flow của Authentications:`

Sign up: 
    - Lấy signup data từ request
    - Kiểm tra username tồn tại chưa
    - Nếu chưa:
        + Mã hóa password
        + Tạo user mới trong db
        + return 204 (No content)

Sign in: 
    - Lấy signin data từ request (username + password)
    - Lấy thông tin user từ data --> lấy password để so sánh
    - Nếu password khớp: 
        + Tạo accessToken bằng JWT
        + Tạo refresh token
        + Tạo session mới trong db để lưu refresh token
        + Trả refresh token về trong cookie
        + return 200 có kèm accessToken

Sign out:
    - Lấy refresh token từ cookie
    - Nếu tồn tại token:
        + Xóa refresh token đó trong db (bảng session)
        + Xóa cookie

=================================================================================================
Flow của Authorization:

- Lấy token từ header
- Xác nhận token có hợp lệ không
    + Lấy thông tin user
    + Trả user về trong request
    + Chuyển đến hành động tiếp theo (middleware khác / vào api)

=================================================================================================
middleware (server.js)

- Các private routes được đặt sau app.use(protectedRoute)
    + Trong protectedRoute --> thực hiện Authorization

=================================================================================================
Socket IO
- Giữ kết nối giữa frontend và backend
- Cần có 1 instance ở frontend và 1 instance ở backend
- Từ khóa: 
    + socket.on = nghe/nhận sự kiện
    + socket.emit = gửi/phát sự kiện
- Setup SocketIO (cho backend)
    + npm i socket.io
    + tạo src/socket/index.js để cấu hình cho socket
    + chỉnh sửa cấu hình trong server.js
    + socket/index.js:
        * tạo socketMiddleware để bảo vệ socket & use middleware 
        * viết hàm emit sự kiện online/offline
    + messageController:
        * viết hàm emit sự kiện new message (controller & helper)

=================================================================================================
Flow theo dõi trạng thái tin nhắn để biết khi nào tin nhắn được đọc
- Khi user mở một conversation:
    + backend: gọi API markAsSeen
        * cập nhật dữ liệu: thêm user vào danh sách seenBy trong conversation
        * reset unreadCounts[user] = 0 (người này không có tin nào chưa đọc)
        * phát socket event rằng "user này đã đọc tin nhắn rồi"
    + frontend: sau khi backend api trả về:
        * cập nhật conversation trong store để UI thay đổi trạng thái
    + khi có tin nhắn mới: nếu tin nhắn thuộc đúng cuộc hội thoại user đang mở --> gọi markAsSeen ngay
